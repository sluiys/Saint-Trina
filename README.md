Saint Trina is a local pre-commit hook that statically analyzes staged files to prevent the leakage of sensitive data and hardcoded tokens into version control.

## Installation

Run the following command at the root of any Git repository to inject the pre-commit hook:

```bash
curl -sSL https://raw.githubusercontent.com/sluiys/Saint-Trina/main/install.rb | ruby
```
## Architecture and Detection Scope

The scanner intercepts the git commit process. If a violation is detected, the commit is aborted automatically. The core engine scans for:

    AWS Access Keys

    Cloud provider tokens (GitHub, Stripe, Google Cloud)

    Hardcoded secrets in standard variable assignments

## Extreme Risk Lockdown

Files designated as inherent security risks (e.g., .env, .pem, id_rsa) are rejected outright prior to line-by-line evaluation.

## Bypass Rules

To allow false positives or deliberate mock data in your commits, use the following inline directives:

    Line Bypass: Append saint-trina:ignore-line as a comment on the flagged line.

    File Bypass: Add saint-trina:ignore-file on line 1 to skip scanning the entire file.

## Uninstallation

To remove the hook from a repository, delete the file generated by the installer:

```bash
rm .git/hooks/pre-commit
```
